چکیده

این پروژه یک سامانه وب برای آموزش زبان انگلیسی است که شامل وب‌سایت کاربری و پنل مدیریت می‌باشد. کاربران می‌توانند دوره‌ها را مرور و ثبت‌نام کنند؛ مدیر سیستم محتوای آموزشی (اساتید، پست‌ها، دوره‌ها)، تعاملات (نظرات، امتیازات، پیام‌ها) و گزارش‌های تحلیلی (ثبت‌نام کاربران، عملکرد اساتید) را مدیریت می‌کند. معماری پروژه مبتنی بر PHP و MySQL با استفاده از PDO و ساختار ساده و قابل نگهداری است. در این پایان‌نامه، نیازمندی‌ها، معماری، طراحی پایگاه‌داده، امنیت، تست، استقرار، مقیاس‌پذیری، و مسیر توسعه آینده تشریح شده است.

1) مقدمه و بیان مسئله

نیاز به بستری سبک، RTL، قابل استقرار روی XAMPP برای مدیریت دوره‌های زبان انگلیسی، ثبت‌نام کاربران، و گزارش‌گیری مدیریتی. هدف، ارائه تجربه یکپارچه برای کاربر و ادمین و پوشش جریان‌های کلیدی است.

2) مرور کلی سیستم

  • بخش کاربری: صفحه اصلی، فهرست و پیش‌نمایش دوره‌ها، اساتید، پست‌ها، ثبت‌نام در دوره، ورود/ثبت‌نام.
  • بخش مدیریت: مدیریت کاربران، اساتید، پست‌ها، نظرات، امتیازات، پیام‌ها، گزارش ثبت‌نام‌ها و گزارش اساتید.
  • قابلیت‌های کلیدی: جدول واسط ثبت‌نام‌ها با جلوگیری از تکراری، گزارش‌های تحلیلی، UI یکپارچه، فونت مشترک.

کارهای مرتبط

پلتفرم‌های آموزش زبان نظیر Coursera/Udemy (دوره‌های عمومی) و Duolingo/Memrise (یادگیری تعاملی) هریک بر بخشی از نیازها تمرکز دارند. سامانه حاضر با رویکرد سبک و قابل‌استقرار محلی (XAMPP) و تمرکز بر مدیریت محتوای بومی، گزارش‌گیری ادمین و ثبت‌نام دوره‌ها برای سناریوهای سازمانی/آموزشگاهی مناسب‌سازی شده است. نسبت به CMSهای عمومی، ساختار دیتابیس اختصاصی (enrollments/ratings/comments) و گزارش‌های تحلیلی دقیق بر کاربران/اساتید ارائه می‌کند.

  • مزیت‌ها: نصب سریع، سادگی نگهداری، شخصی‌سازی آسان UI/داده‌ها، گزارش‌های مدیریتی.
  • محدودیت‌ها: نبود LMS کامل (آزمون/تکلیف/گواهی) که در توسعه آینده پیش‌بینی شده است.

روش تحقیق

روش تحقیق مبتنی بر مهندسی نرم‌افزار تکرارشونده است: گردآوری نیازمندی‌ها، طراحی معماری و دیتابیس، پیاده‌سازی افزایشی (ویژگی‌های کاربر/ادمین/گزارش‌ها)، و اعتبارسنجی از طریق تست کارکردی. انتخاب فناوری‌ها بر اساس در دسترس بودن، سازگاری با محیط‌های آموزشی، و هزینه نگهداری انجام شده است. معیارهای ارزیابی شامل کارایی (پاسخگویی کوئری‌ها)، امنیت پایه (PDO/Hash)، و کیفیت UI (همسانی سبک و ریسپانسیو) بوده‌اند.

3) فناوری‌ها و معماری

  • Backend: PHP 8.2، PDO، الگوی فایل‌محور.
  • Database: MariaDB/MySQL، ایندکس‌ها و قیود خارجی.
  • Frontend: HTML/CSS/JS، Bootstrap، استایل‌های app.css و style.css، فونت Vazir.
  • ساختار: includes/database.php برای اتصال، صفحات PHP برای منطق و نما، includes/media.php برای مسیر تصاویر.

4) طراحی پایگاه‌داده

جداول: users، instructors، courses، enrollments، posts، comments، ratings، contact_messages. کلید یکتا در enrollments (user_id, course_id)، قیود خارجی با حذف آبشاری، و ایندکس‌های پیشنهادی برای کارایی.

جدولستون‌های کلیدی
usersid, username, email, password, role, created_at
coursesid, title, description, category, level, price, status, image_url, instructor_id
enrollmentsid, user_id, course_id, status, enrolled_at (UNIQUE user_id,course_id)
instructorsid, name, bio, photo_url, created_at
postsid, title, content, image_url, instructor_id, created_at
commentsid, user_id, post_id, content, status, created_at
ratingsid, user_id, instructor_id, rating, created_at
contact_messagesid, name, email, message, created_at

5) ویژگی‌ها و جریان‌های کاری

  • ثبت‌نام دوره: enroll.php → process_enrollment.php (درج یا فعال‌سازی دوباره).
  • گزارش کاربران: admin_enrollments_report.php + جزئیات user_enrollments.php.
  • گزارش اساتید: admin_instructors_report.php + جزئیات instructor_overview.php.

6) امنیت و محرمانگی

Prepared Statements، هش رمز عبور، کنترل نقش ادمین در auth_check.php، اعتبارسنجی آپلود، مدیریت سشن‌ها.

7) تصمیمات پیاده‌سازی

  • جدول واسط با UNIQUE و هندلینگ Duplicate Key (1062).
  • اصلاح ORDER BY برای MariaDB بدون NULLS LAST (استفاده از زیرکوئری).
  • یکپارچه‌سازی UI ادمین با استایل سایت و فونت مشترک.
  • تابع resolveCourseImage برای نگاشت تصاویر PNG/JPG.

8) تست و کیفیت

تست دستی سناریوهای کلیدی (ثبت‌نام، گزارش‌ها، تصاویر)، مدیریت استثناهای PDO، و اصلاح ناسازگاری‌های SQL.

9) استقرار و پیکربندی

  • XAMPP روی Windows (Apache + PHP + MariaDB).
  • Import دیتابیس: english_language_learning.sql + اجرای admin/create_enrollments_table.sql.
  • تنظیم includes/database.php و اجرای admin/setup.php.

10) کارایی و مقیاس‌پذیری

  • ایندکس‌های کلیدی، Queryهای هدفمند.
  • پیشنهاد: صفحه‌بندی، Cache گزارش‌ها، CDN برای استاتیک‌ها.

11) توسعه آینده

  • پرداخت آنلاین، پنل کاربری پیشرفته، گواهی، آزمون و تکلیف، فروم.
  • چندزبانه‌سازی، نقش‌های پیشرفته، نوتیفیکیشن‌ها.
  • Docker، CI/CD، مانیتورینگ، بکاپ خودکار.
  • افزودن CSRF، محدودسازی نرخ، سیاست‌های آپلود سخت‌گیرانه‌تر.

12) محدودیت‌ها و ریسک‌ها

عدم ORM، احراز هویت ساده، نیاز به سیاست‌های قوی‌تر آپلود و تست‌های خودکار بیشتر.

13) نتیجه‌گیری

معماری ساده و توسعه‌پذیر با UI یکپارچه نیازهای سامانه آموزشی سبک را برآورده می‌کند و با افزودن ماژول‌های پرداخت، محتوای تعاملی و DevOps آماده تولید خواهد بود.
